# Coze 浏览器插件代理服务部署方案

## 一、问题说明

浏览器插件在拉取 Coze API 时，需要通过本地代理服务解决跨域问题。客服同事的电脑需要启动这个代理才能正常使用插件。

当前架构：
```
浏览器插件 → localhost:8888 代理 → api.coze.cn (HTTPS)
```

---

## 二、方案对比

| 方案 | 复杂度 | 是否需要安装 Node.js | 客服操作难度 |
|------|--------|---------------------|--------------|
| 方案A：批处理文件 | 低 | ✅ 需要 | 双击运行（需确保 Node.js 已安装）|
| 方案B：打包成 .exe | 中 | ❌ 不需要 | 双击运行 |
| 方案C：开机自启动服务 | 高 | ❌ 不需要 | 无需操作，开机自动启动 |

---

## 三、推荐方案：方案B（打包成 .exe）

### 3.1 方案说明

将代理服务打包成单个可执行文件，客服无需安装任何环境，双击即可运行。

### 3.2 实施步骤

**第1步：安装打包工具**

```bash
npm install -g pkg
```

**第2步：打包项目**

```bash
cd C:\Users\Administrator\.openclaw\workspace\company-website
pkg server.js --output coze-proxy.exe
```

这会生成 `coze-proxy.exe` 文件（约 30MB）。

**第3步：分发给客服**

把 `coze-proxy.exe` 发给客服，告知他们：
1. 复制到电脑的任意位置（如桌面）
2. **双击运行**
3. 保持窗口开启（最小化即可）

### 3.3 客服使用流程

```
1. 双击 coze-proxy.exe
2. 看到窗口显示 "Server running on http://localhost:8888"
3. 打开Chrome浏览器，使用插件
4. 用完后关闭窗口即可
```

---

## 四、最终实施方案：Node.js 便携版

将 Node.js 可执行文件直接打包到项目文件夹中，客服无需安装任何软件。

### 4.1 客服操作步骤

```
1. 解压项目文件夹（包含 node.exe）
2. 双击运行"启动代理.bat"
3. 保持黑色窗口开启
4. 打开浏览器使用插件
```

### 4.2 验证是否正常运行

打开浏览器访问：http://localhost:8888/api/test-cors

如果显示 `{"ok":true,"message":"CORS works!"}` 则表示代理服务正常运行。

---

## ⚠️ 方案B和C说明

方案B（pkg打包）和方案C（开机自启动）暂不可行，因为：
- 当前打包工具 pkg 与 Node v24 不兼容
- 需要在 Node 版本较低的环境（如 Node 18）下打包

**如果后续需要方案B**，可以在安装了 Node 18 的电脑上执行：
```bash
pkg server.js --output coze-proxy.exe --targets node18-win-x64
```
