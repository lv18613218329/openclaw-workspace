## 三、技术选型分析

### 2.1 渲染引擎对比

| 渲染引擎 | 类型 | 图形分割 | 物理化学图形 | 跨平台 | 性能 |
|---------|------|---------|------------|-------|------|
| **Skia** | 2D 矢量 | ✅ 原生 Path Op | ⭐⭐⭐⭐ | 全平台 | 高 |
| **Canvas 2D** | Web 2D | ⚠️ 需 clipper.js | ⭐⭐⭐ | Web/Desktop | 中 |
| **Direct2D** | Windows 2D | ✅ | ⭐⭐⭐⭐ | 仅 Windows | 高 |
| **Cairo** | 2D 矢量 | ⚠️ 有限 | ⭐⭐⭐ | 全平台 | 中 |
| **OpenGL/Vulkan** | 3D | ❌ 需自己实现 | ⭐⭐⭐⭐⭐ | 全平台 | 最高 |
| **Three.js** | Web 3D | ❌ | ⭐⭐⭐⭐⭐ | Web/Desktop | 中高 |

### 2.2 物理化学图形的渲染需求分析

#### 化学 3D 分子结构 → 需要 3D 能力

```
选项 A：Skia（2D）+ 3D 扩展
- 2D 结构式：Skia 完美支持
- 3D 分子模型：需要引入 3D 引擎（如 HelixToolkit / OpenTK）

选项 B：统一 3D 引擎（OpenGL/Vulkan/Three.js）
- 可以渲染所有内容
- 但 2D 编辑交互复杂度高
```

#### 物理电路图 → 需要 2D 矢量 + 符号库

```
2D 矢量引擎足够
关键是元器件符号库的设计
```

#### 数学公式 → 需要 LaTeX 渲染

```
选项：MathJax / KaTeX（Web）
      C# 可用 MathJax 通过 WebView 嵌入
      或者 SkiaSharp 自绘公式
```

### 2.3 推荐技术路线

#### 路线 A：纯 2D 方案（推荐起步）

```
Avalonia + Skia（主渲染引擎）

适用场景：
✅ 化学结构式（2D）
✅ 物理电路图、力学图、光路图
✅ 波形图、实验装置图
⚠️ 3D 分子模型不支持

技术栈：
- UI 框架：Avalonia
- 2D 渲染：Skia（通过 SkiaSharp.Views.Avalonia）
- 公式渲染：嵌入 WebView + KaTeX，或自绘
- 图形分割：SKPath.Op() 原生支持
```

#### 路线 B：2D + 3D 混合方案（完整功能）

```
Avalonia + Skia（2D）+ Silk.NET（3D）

适用场景：
✅ 所有 2D 图形
✅ 3D 分子模型
✅ 物理仿真动画

技术栈：
- UI 框架：Avalonia
- 2D 渲染：Skia
- 3D 渲染：Silk.NET（OpenGL/Vulkan 绑定）
- 分子可视化：自定义或集成 OpenBabel
```

#### 路线 C：Web 混合方案

```
Electron + tldraw（2D）+ Three.js（3D）

适用场景：
✅ 开发速度快
✅ Web 生态丰富
⚠️ 桌面端包体大
⚠️ 移动端体验一般
```

### 2.4 最终推荐

| 阶段 | 方案 | 理由 |
|-----|------|------|
| **MVP（最小可行产品）** | Avalonia + Skia | 快速验证核心功能，图形分割原生支持 |
| **完整版** | Avalonia + Skia + Silk.NET | 增加 3D 分子模型能力 |

#### 3.2.2 图形分割引擎

```csharp
public class BooleanEngine
{
    /// <summary>
    /// 使用 Skia Path Operations 实现图形分割
    /// </summary>
    public List<SKPath> BooleanOperation(
        SKPath pathA, 
        SKPath pathB, 
        BooleanOp operation)
    {
        var result = new List<SKPath>();
        
        switch (operation)
        {
            case BooleanOp.Union:
                var union = new SKPath();
                union.Op(pathA, pathB, SKPathOp.Union);
                result.Add(union);
                break;
                
            case BooleanOp.Intersection:
                var intersect = new SKPath();
                intersect.Op(pathA, pathB, SKPathOp.Intersect);
                result.Add(intersect);
                break;
                
            case BooleanOp.Difference:
                var diff = new SKPath();
                diff.Op(pathA, pathB, SKPathOp.Difference);
                result.Add(diff);
                break;
                
            case BooleanOp.Split:
                // 分割：返回交集部分 + 差集部分
                var inter = new SKPath();
                inter.Op(pathA, pathB, SKPathOp.Intersect);
                var diff1 = new SKPath();
                diff1.Op(pathA, pathB, SKPathOp.Difference);
                var diff2 = new SKPath();
                diff2.Op(pathB, pathA, SKPathOp.Difference);
                result.AddRange(new[] { inter, diff1, diff2 });
                break;
        }
        
        return result;
    }
}

public enum BooleanOp
{
    Union,         // 并集
    Intersection,  // 交集
    Difference,    // 差集
    Split          // 分割
}

public class ConnectionEngine
{
    public Connection CreateConnection(
        AnchorPoint source, 
        AnchorPoint target,
        ConnectionStyle style)
    {
        var connection = new Connection
        {
            Source = source,
            Target = target,
            Style = style
        };
        
        // 计算路径
        connection.Path = CalculatePath(source, target, style);
        
        return connection;
    }
    
    private SKPath CalculatePath(
        AnchorPoint source, 
        AnchorPoint target,
        ConnectionStyle style)
    {
        return style switch
        {
            ConnectionStyle.Straight => CreateStraightPath(source, target),
            ConnectionStyle.Polyline => CreatePolylinePath(source, target),
            ConnectionStyle.Curved => CreateBezierPath(source, target),
            ConnectionStyle.Orthogonal => CreateOrthogonalPath(source, target),
            _ => throw new ArgumentException()
        };
    }
}

└── 优点：渲染效果最好，社区支持完善

方案 B：Skia 自绘
├── 解析 LaTeX 子集
├── 使用 Skia 绘制公式
├── 完全离线，无依赖
└── 缺点：开发量大，LaTeX 语法复杂
```

## 七、关键技术验证

### 6.1 图形分割验证

```csharp
// 验证 Skia Path Operations 是否满足需求
public void TestShapeSplitting()
{
    // 创建一个矩形
    var rect = new SKPath();
    rect.AddRect(new SKRect(0, 0, 100, 100));
    
    // 创建一条分割线
    var line = new SKPath();
    line.MoveTo(50, -10);
    line.LineTo(50, 110);
    line.LineTo(60, 110);
    line.LineTo(60, -10);
    line.Close();
    
    // 分割
    var left = new SKPath();
    left.Op(rect, line, SKPathOp.Difference);
    
    var right = new SKPath();
    var lineShifted = new SKPath();
    lineShifted.AddRect(new SKRect(50, 0, 60, 100));
    right.Op(rect, lineShifted, SKPathOp.Intersect);
    
    // 结果：两个独立图形，可独立旋转
}
```
