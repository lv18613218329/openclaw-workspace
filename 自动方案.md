### 实现方式

**1. 创建采集 Skill**

我可以帮你创建一个专门的 skill，功能包括：
- 自动遍历全国31个省市的政府采购信息
- 提取项目详情并去重
- 增量更新（只保存新增项目）
- 支持多种关键词配置

**2. 配置 Cron 定时任务**

```json
{
  "name": "双高建设数据采集",
  "schedule": { "kind": "cron", "expr": "0 9 * * *" },  // 每天早上9点
  "payload": { 
    "kind": "agentTurn", 
    "message": "运行双高建设采购信息采集任务，遍历全国省市，增量更新数据"
  },
  "sessionTarget": "isolated",
  "delivery": { "mode": "announce" }  // 完成后通知你
}
```

## 📊 数据存储方案

### 方案A：CSV 文件（简单）
- 每天追加新数据
- 适合数据量小、快速上手
- 文件：`双高建设_全国_[日期].csv`

### 方案B：SQLite 数据库（推荐）
```sql
CREATE TABLE procurement_items (
  id INTEGER PRIMARY KEY,
  title TEXT,
  url TEXT UNIQUE,  -- 用于去重
  publish_date DATE,
  buyer TEXT,
  agent TEXT,
  region TEXT,
  type TEXT,
  created_at DATETIME
);
```

### 方案C：飞书多维表格（协作友好）
- 你已有飞书集成
- 可以直接推送到飞书多维表格
- 方便团队协作和筛选

## 🔧 技术实现细节

### 关键挑战 & 解决方案

| 挑战 | 解决方案 |
|------|----------|
| 网站反爬 | 使用 OpenClaw 管理的浏览器，模拟正常访问 |
| 增量更新 | 用 URL 或项目编号作为唯一键去重 |
| 大量省市 | 分批次查询，每次间隔几秒，避免被封 |
| 数据解析 | 使用 ARIA snapshot 提取结构化数据 |
