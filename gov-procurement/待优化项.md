# 待优化项

## 已知问题

### 1. 地区字段提取率低

**现状**：地区字段提取率仅 37%（158条中99条为空）

**原因**：
- 地区信息在页面上的格式不统一
- 部分公告未标注地区
- 正则表达式匹配不够全面

**影响**：无法按地区筛选数据

**优先级**：中

**改进方案**：
```javascript
// 方案1：扩展地区匹配规则
const REGION_PATTERNS = [
  /\|\s*(四川|北京|...)\s*\|/,
  /地区[：:]\s*([^|\s]+)/,
  /省份[：:]\s*([^|\s]+)/
];

// 方案2：从采购人名称推断地区
// 例如："四川大学" → 地区="四川"
```

### 2. 部分采购人未提取

**现状**：14条记录采购人为空

**原因**：
- 页面格式变化
- 采购人信息位置不固定

**影响**：无法进行学校关键词过滤

**优先级**：高

**改进方案**：
- 增加采购人信息的匹配模式
- 从详情页提取采购人信息

**原因**：未实现翻页逻辑

**影响**：热门关键词数据不完整

**优先级**：高

**改进方案**：
```javascript
// 检测总页数

const totalCount = parseInt(totalMatch[1]);
const totalPages = Math.ceil(totalCount / 20);

// 翻页采集
for (let page = 1; page <= totalPages; page++) {
  const url = buildSearchUrl(keyword, startDate, endDate, page);
  // ...
}
```

### 4. 无详情页采集

**现状**：仅采集列表页信息

**缺失字段**：
- 预算金额
- 采购人联系方式
- 代理机构联系方式
- 项目详细内容

**优先级**：中

**改进方案**：
```javascript
// 访问详情页
const detailUrl = item.url;
await navigate(detailUrl);
const detailSnapshot = await snapshot();

// 提取详细信息
item.budget = extractBudget(detailSnapshot);
```

## 技术债务

### 代码层面

1. **缺少错误处理**：网络超时、页面加载失败处理不完善
2. **缺少日志系统**：采集过程难以追溯
3. **配置硬编码**：关键词、时间范围应支持配置文件

### 架构层面

1. **单进程采集**：大量关键词时效率低
2. **无断点续传**：采集中断需重新开始
3. **无监控告警**：采集失败无通知
