# 使用教程

## 环境要求

| 组件 | 要求 |
|------|------|
| Node.js | v18+ |
| OpenClaw | 已安装并运行 |
| 浏览器 | Chrome/Edge，已安装 OpenClaw Browser Relay 扩展 |

---

## 部署步骤

### 1. 获取项目文件

将整个 `gov-procurement` 目录复制到目标机器的 OpenClaw workspace：

```bash
# Windows
复制到: C:\Users\{用户名}\.openclaw\workspace\gov-procurement

# Linux/Mac
复制到: ~/.openclaw/workspace/gov-procurement
```

### 2. 安装依赖

```bash
cd ~/.openclaw/workspace/gov-procurement
npm install
```

### 3. 初始化数据库

```bash
node init-db.js
```

### 4. 验证安装

```bash
node show-data.js
```

如果输出数据库表结构，说明安装成功。

## 运行采集

### 基本用法

```bash
node crawl-v3.js
```

### 前置条件

1. **启动 OpenClaw Gateway**

```bash
openclaw gateway start
```

2. **打开浏览器并激活扩展**

- 打开 Chrome/Edge 浏览器
- 点击 OpenClaw Browser Relay 扩展图标
- 确保徽章显示 **ON**（绿色）

3. **访问政府采购网**

- 在浏览器中打开 https://search.ccgp.gov.cn/
- 确保页面正常加载

## 配置修改

### 修改采集关键词

编辑 `crawl-v3.js` 中的 `ALL_KEYWORDS` 变量：

```javascript
const ALL_KEYWORDS = {
  "政策项目类": ["双优校建设", "双高建设"],
  // 添加更多...
};
```

### 修改时间范围

编辑 `crawl-v3.js` 中的时间设置：

```javascript
// 最近30天（默认）
startDate.setDate(startDate.getDate() - 30);

// 改为最近7天
startDate.setDate(startDate.getDate() - 7);
```

### 修改学校关键词

编辑 `crawl-v3.js` 中的 `SCHOOL_KEYWORDS` 变量：

```javascript
const SCHOOL_KEYWORDS = ['学校', '中心', '学院', '中学', '高中', '集团', '大学', '小学'];
// 添加更多关键词...
```

## 定时任务

### 使用 OpenClaw Cron

在 OpenClaw 中配置定时任务：

```javascript
// 每天早上6点执行
{
  "schedule": { "kind": "cron", "expr": "0 6 * * *" },
  "payload": {
    "kind": "agentTurn",
    "message": "执行政府采购信息采集：cd ~/.openclaw/workspace/gov-procurement && node crawl-v3.js"
  }
}
```

### 使用系统 Cron（Linux）

```bash
# 编辑 crontab
crontab -e

# 添加任务（每天6点执行）
0 6 * * * cd ~/.openclaw/workspace/gov-procurement && node crawl-v3.js
```

### 使用 Windows 任务计划

1. 打开"任务计划程序"
2. 创建基本任务
3. 设置触发器：每天 06:00
4. 操作：启动程序
   - 程序：`node`
   - 参数：`crawl-v3.js`
   - 起始位置：`C:\Users\{用户名}\.openclaw\workspace\gov-procurement`

## 常见问题

### Q1: 采集返回"找到0条"

**可能原因**：
1. 浏览器扩展未激活
2. URL格式错误
3. 网络问题

**解决方案**：
1. 检查浏览器扩展徽章是否为绿色
2. 确认政府采购网可以正常访问
3. 查看控制台错误信息

### Q2: 浏览器快照返回"Internal Server Error"

**可能原因**：政府采购网服务器过载

**解决方案**：
1. 等待几分钟后重试
2. 手动刷新浏览器页面
3. 更换访问时间（避开高峰期）

### Q3: 数据重复

**原因**：多次采集相同关键词

**解决方案**：
- 程序使用 `INSERT OR IGNORE` 自动去重
- 重复URL不会重复入库

### Q4: 如何清空数据库重新采集

```bash
# 方法1：删除数据库文件
rm procurement.db
node init-db.js

# 方法2：运行清理脚本
node clean-invalid.js  # 只清理异常数据
```

## 数据库操作

### 连接数据库

```bash
sqlite3 procurement.db
```

### 常用查询

```sql
-- 查看总数
SELECT COUNT(*) FROM procurement_items;

-- 按关键词统计
SELECT keyword, COUNT(*) as count 
FROM procurement_items 
GROUP BY keyword 
ORDER BY count DESC;

-- 按地区统计
SELECT region, COUNT(*) as count 
FROM procurement_items 
WHERE region IS NOT NULL
GROUP BY region 
ORDER BY count DESC;

-- 查看最近采集
SELECT * FROM procurement_items 
ORDER BY created_at DESC 
LIMIT 10;

-- 查看采集日志
SELECT * FROM search_logs 
ORDER BY search_time DESC 
LIMIT 20;
```
