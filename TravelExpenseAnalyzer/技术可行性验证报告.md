## 三、特殊处理项

### 3.1 在途酒店订单摘要解析

**格式**：
```
{城市} {酒店名称}-{房型} {入住日期} ~ {离店日期} {出行人姓名}
```

**示例**：
```
平顶山 维也纳酒店(平顶山卫东区万达广场店)-标准大床房 2026-01-30 ~ 2026-01-31 舒锐
眉山 伊菲精品酒店(三苏祠店)-雅致大床房 2026-01-29 ~ 2026-01-30 刘俊江
```

**解析正则**：
```javascript
/^(.+?)\s+(.+?)-(.+?)\s+(\d{4}-\d{2}-\d{2})\s*~\s*(\d{4}-\d{2}-\d{2})\s+(.+)$/
```

**提取字段**：
| 组号 | 字段 |
|------|------|
| 1 | 城市 |
| 2 | 酒店名称 |
| 3 | 房型 |
| 4 | 入住日期 |
| 5 | 离店日期 |
| 6 | 出行人姓名 |

### 3.2 在途机票订单摘要

**格式**：
```
{出发地}-{目的地} {日期} {时间}起飞 {航班号}{舱位信息} {出行人}
```

**示例**：
```
济南-成都 2026-02-05 16:50起飞 SC8831经济舱2.6折  王齐
```

**说明**：机票数据已有出发地、目的地、行程开始时间字段，摘要可作为补充验证。

### 3.3 在途用车订单摘要

**格式**：
```
{起点}-{终点}  {车型} {日期} {时间}出发 {出行人}
```

**示例**：
```
湘潭站(出站口)西南侧-湘潭生物机电学校  经济型 2026-01-27 09:42出发 宋宝峰
```

**注意**：用车订单的出发地/目的地字段可能为空，需从摘要解析。

### 3.4 在途火车票订单摘要

**格式**：
```
{出发地}-{目的地} {车次}{座位类型}{日期} {时间}出发  {出行人}
```

**示例**：
```
遂宁-成都东 D631无座2026-01-30 15:14出发  冉崇洋
```

**说明**：火车票数据已有出发地、目的地字段。

## 五、验证结论

### 5.1 技术方案可行性

**结论：✓ 技术方案可行，可以实施**

### 5.2 支持理由

1. **必需字段完整**：出差人、费用发生日期、填报日期等核心字段所有数据源均有
2. **金额字段可用**：交通费、住宿费金额字段完整，可直接使用
3. **时间格式统一**：各平台时间格式一致（YYYY-MM-DD HH:mm:ss）
4. **摘要格式固定**：在途订单摘要格式规律，可通过正则解析
5. **留空字段合理**：税额、补助等字段源数据无，符合业务实际情况

### 5.3 处理建议

| 项目 | 建议 |
|------|------|
| 在途服务费 | 不单独处理，作为附加费用 |
| 美团餐饮 | 暂不处理，需确认是否纳入 |

| 美团打车区县 | 从地点文本解析，或使用城市级别 |
