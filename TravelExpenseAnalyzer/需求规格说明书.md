## 一、项目概述

### 1.1 项目背景

企业需要追溯和还原员工的出行轨迹和费用情况，核实员工是否存在以下违规行为：
1. 同一时间出现在多个地点
2. 同一时间产生多笔费用
3. 地点信息与实际情况不符

目前数据分散在多个差旅平台导出的Excel文件中，格式不统一，人工核对工作量大且容易遗漏。需要开发工具实现数据整合和异常自动检测。

### 1.2 项目目标

1. **数据整合**：将滴滴、美团、在途三个平台的差旅数据整合成统一格式
2. **异常检测**：自动识别时间重叠、地点冲突等异常情况
3. **可视化标记**：异常数据在Excel中标红，警示数据标黄
4. **数据核对**：提供完整的核对机制，确保数据完整性

### 1.3 项目范围

| 范围内 | 范围外 |
|--------|--------|
| 滴滴用车订单数据处理 | 其他打车平台数据 |
| 美团商旅订单数据处理（打车、酒店、机票、餐饮） | 美团外卖等非商旅数据 |
| 在途差旅订单数据处理 | 其他差旅平台数据 |
| 时间重叠异常检测 | 费用金额异常检测（无标准） |
| 地点冲突异常检测 | 发票真伪验证 |
| 多地行程警示 | 差旅审批流程 |
| 数据核对报告生成 | 自动上报系统 |

### 3.2 功能详细说明

#### 3.2.1 F01 滴滴数据导入

**输入**：滴滴用车订单Excel文件

**处理规则**：
1. 读取文件第一个Sheet
2. 跳过空行和表头行
3. 过滤订单状态为"退款"的记录
4. 提取有效字段

**输出**：滴滴数据记录数组

**数据过滤条件**：
- 订单状态 = "支付"（排除退款订单）

#### 3.2.2 F02 美团数据导入

**输入**：美团商旅订单Excel文件（多Sheet）

**处理规则**：
1. 遍历所有Sheet
2. 识别有效数据Sheet（数据量 > 0）
3. 根据Sheet名称判断业务类型
4. 合并所有有效Sheet数据

**有效Sheet识别**：
| Sheet名称 | 业务类型 |
|-----------|----------|
| 打车 | 用车 |
| 国内酒店 | 住宿 |
| 国内机票 | 机票 |
| 用券买单 | 餐饮 |
| 买单 | 餐饮 |

**输出**：美团数据记录数组（含业务类型标识）

#### 3.2.3 F03 在途数据导入

**输入**：在途差旅订单Excel文件

**处理规则**：
1. 读取文件第一个Sheet
2. 根据业务类型字段判断订单类型
3. 解析订单摘要字段提取酒店信息

**订单摘要解析规则**：
```
格式：{城市} {酒店名称}-{房型} {入住日期} ~ {离店日期} {出行人姓名}

正则表达式：/^(.+?)\s+(.+?)-(.+?)\s+(\d{4}-\d{2}-\d{2})\s*~\s*(\d{4}-\d{2}-\d{2})\s+(.+)$/

提取字段：
- 城市：第1组
- 酒店名称：第2组
- 房型：第3组
- 入住日期：第4组
- 离店日期：第5组
- 出行人姓名：第6组
```

**输出**：在途数据记录数组

#### 3.2.4 F04 数据字段映射

**输入**：各平台原始数据

**处理规则**：根据映射规则将源字段转换为目标字段

**映射规则详见**：第四章 字段映射规则

**输出**：统一格式的数据记录

#### 3.2.5 F05 数据清洗

**清洗项目**：

| 清洗项 | 规则 | 处理方式 |
|--------|------|----------|
| 出差人为空 | 字段为null或空字符串 | 记录警告，跳过该条 |
| 日期格式 | Excel序列号或日期字符串 | 统一转换为YYYY-MM-DD格式 |
| 金额格式 | 字符串转数值 | 转换失败则置0 |
| 时间格式 | 日期时间字符串 | 统一转换为Date对象 |
| 地点格式 | 地址文本 | 保留原始格式，用于显示 |

#### 3.2.6 F06 时间重叠检测

**检测目标**：同一人在同一时间段内出现在不同地点

**检测精度**：精确到小时

**检测算法**：
```
1. 按出差人分组
2. 对每人的记录按开始时间排序
3. 遍历比较每对记录：
   - 记录A的时间区间：[A.开始时间, A.结束时间]
   - 记录B的时间区间：[B.开始时间, B.结束时间]
   - 如果 A.开始时间 < B.结束时间 且 B.开始时间 < A.结束时间
   - 且 A.地点 ≠ B.地点
   - 则标记为时间重叠异常
```

**时间区间定义**：

| 数据类型 | 开始时间 | 结束时间 | 备注 |
|----------|----------|----------|------|
| 打车（滴滴） | 开始计费时间 | 结束计费时间 | 精确到分钟 |
| 打车（美团） | 行程开始时间 | 行程到达时间 | 精确到分钟 |
| 酒店 | 入住时间 | 离店时间 | 可能跨多天 |
| 机票 | 起飞时间 | 到达时间 | 精确到分钟 |
| 无明确结束时间 | 费用发生日期 00:00 | 费用发生日期 23:59 | 按整天处理 |

**异常标记**：整行标红

#### 3.2.7 F07 地点冲突检测

**检测目标**：同一人在同一时间在不同地点

**检测精度**：
- 打车、酒店：区县级
- 机票：城市级（因机票数据只有城市信息）

**检测算法**：
```
1. 按出差人分组
2. 对每人在同一时间区间内的记录提取地点
3. 比较地点是否冲突：
   - 如果都是区县级：比较区县是否相同
   - 如果包含机票（城市级）：比较城市是否相同
   - 同一城市内不同区县不算冲突
```

**地点提取规则**：

| 数据类型 | 地点字段 | 精度 | 提取规则 |
|----------|----------|------|----------|
| 打车（滴滴） | 出发区县、到达区县 | 区县 | 取非空值 |
| 打车（美团） | 用户选择上车地点 | 需解析 | 从文本解析城市+区县 |
| 酒店 | 酒店地址 | 区县 | 从地址解析城市+区县 |
| 机票 | 出发城市名称、到达城市名称 | 城市 | 直接使用城市名 |

**异常标记**：整行标红

#### 3.2.8 F08 多地行程警示

**检测目标**：一人一天内出现在多个城市

**检测算法**：
```
1. 按出差人分组
2. 按日期聚合每人的记录
3. 统计当天涉及的城市数量
4. 如果城市数量 > 1，标记为多地行程警示
```

**说明**：
- 此情况通常为正常出差（如转机、跨城市出差）
- **不算异常**，但需要人工确认
- 用黄色标记以区分

**警示标记**：整行标黄

#### 3.2.9 F09 异常标记

**标记规则**：

| 标记类型 | 条件 | 颜色 | 背景色 | 字体色 |
|----------|------|------|--------|--------|
| 正常 | 无异常 | 无 | 白色 | 黑色 |
| 异常 | 时间重叠或地点冲突 | 红色 | #FF0000 | 白色 |
| 警示 | 多地行程 | 黄色 | #FFFF00 | 黑色 |

**标记方式**：整行标记（所有单元格应用相同样式）

**优先级**：红色（异常）> 黄色（警示）

#### 3.2.10 F10 数据核对

**核对项目**：

| 核对项 | 核对内容 | 通过条件 |
|--------|----------|----------|
| 文件读取 | 所有输入文件成功读取 | 读取条数 > 0 |
| 条数一致 | 输入条数 = 输出条数 | 差值 = 0 |
| 必填字段 | 出差人、费用发生日期非空 | 空值数 = 0 |
| 日期格式 | 日期字段格式正确 | 格式错误数 = 0 |
| 金额格式 | 金额字段为数值 | 非数值数 = 0 |
| 异常覆盖 | 所有记录完成异常检测 | 未检测数 = 0 |

**核对结果**：
- 全部通过：输出"✓ 全部通过"
- 存在问题：输出"✗ 存在问题"并列出问题清单

#### 3.2.11 F11 核对报告生成

**输出文件**：核对报告_{日期}.txt

**报告内容**：
1. 源数据统计（各平台数据量）
2. 数据转换统计（成功/失败数量）
3. 异常检测统计（各类型异常数量）
4. 异常明细（异常人员、时间、地点详情）
5. 数据完整性核对结果
6. 输出文件列表

**报告格式详见**：第五章 核对报告格式

#### 3.2.12 F12 Excel导出

**输出文件**：出差日记账_生成_{日期}.xlsx

**输出格式**：
- 严格按照出差日记账.xlsx模板格式
- 13个字段，顺序与模板一致
- 第一行为表头
- **排序规则**：
  1. 一级排序：出差人（员工姓名）
  2. 二级排序：费用发生日期

**样式要求**：
- 表头：加粗，背景色灰色
- 正常行：无特殊样式
- 异常行：背景红色，字体白色
- 警示行：背景黄色，字体黑色

## 九、数据校验规则

### 9.1 输入数据校验

| 校验项 | 校验规则 | 失败处理 | 严重程度 |
|--------|----------|----------|----------|
| 文件存在 | 文件路径存在且可读 | 终止程序，提示错误 | 严重 |
| Sheet存在 | Sheet名称在文件中存在 | 跳过该Sheet，记录警告 | 一般 |
| 数据行数 | 数据行数 >= 1（不含表头） | 跳过该文件，记录警告 | 一般 |
| 列名匹配 | 必需列名存在于表头 | 终止程序，提示缺少列 | 严重 |

### 9.2 字段数据校验

| 校验项 | 校验规则 | 失败处理 | 严重程度 |
|--------|----------|----------|----------|
| 出差人 | 非空且非空白字符串 | 跳过该条，记录警告 | 一般 |
| 费用发生日期 | 有效日期格式（Excel序列号或日期字符串） | 跳过该条，记录警告 | 一般 |
| 时间字段 | 有效时间格式或可解析 | 时间区间置空，不参与检测 | 一般 |
| 金额字段 | 数值类型或空值 | 金额置0，记录警告 | 一般 |
| 地点字段 | 非空字符串 | 标记为"未知地点"，继续处理 | 一般 |

### 9.3 转换结果校验

| 校验项 | 校验规则 | 通过条件 |
|--------|----------|----------|
| 条数一致 | 输入条数 = 输出条数 | 差值 = 0 |
| 必填字段 | 出差人、费用发生日期非空 | 空值数 = 0 |
| 日期格式 | 日期字段格式正确 | 格式错误数 = 0 |
| 金额格式 | 金额字段为数值或空 | 非数值数 = 0 |
| 异常覆盖 | 所有记录完成异常检测 | 未检测数 = 0 |
