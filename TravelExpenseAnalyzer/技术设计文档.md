# 出差日记账数据整合工具 - 技术设计文档

## 一、项目背景

### 1.1 业务需求
追溯和还原员工的出行轨迹和费用情况，核实员工是否在同一时间出现在多个地点、产生多笔费用的违规行为。如果出现异常数据，需要在Excel中进行标红，以方便人工核对和处理。

### 1.2 数据来源
现有3个Excel文件，来自不同差旅平台导出的员工出差报销明细：

| 序号 | 文件路径 | 说明 |
|------|----------|------|
| 1 | `C:\Users\Administrator\Desktop\商旅订单\商旅订单\用车订单导出...（滴滴）.xlsx` | 滴滴出行用车订单 |
| 2 | `C:\Users\Administrator\Desktop\商旅订单\商旅订单\商旅订单2026.1.1-2026.1.31（美团）.xlsx` | 美团商旅订单（多sheet） |
| 3 | `C:\Users\Administrator\Desktop\商旅订单\商旅订单\差旅订单汇总订单导出 （在途）.xlsx` | 在途差旅订单 |

### 1.3 输出要求
生成符合 `出差日记账.xlsx` 模板格式的Excel文件，异常数据标红，警示数据标黄。

**排序规则**：
1. 一级排序：出差人（员工姓名）- 按字母/拼音顺序
2. 二级排序：费用发生日期 - 按时间先后顺序

## 四、异常检测规则

### 4.1 检测类型定义

| 检测类型 | 定义 | 标记颜色 | 说明 |
|----------|------|----------|------|
| **时间重叠异常** | 同一人在同一时间段内出现在不同地点 | 红色 | 严重异常 |
| **地点冲突异常** | 同一人在同一时间在不同区县 | 红色 | 严重异常 |
| **多地行程警示** | 一天内去了多个城市（出差途中转机等） | 黄色 | 轻微警示，非异常 |

### 4.2 时间重叠检测

#### 4.2.1 检测逻辑
1. 按出差人分组
2. 对每人的所有记录提取时间区间 [开始时间, 结束时间]
3. 检查同一人的任意两条记录是否存在时间区间交集
4. 若存在交集且地点不同 → 标记为时间重叠异常

#### 4.2.2 时间区间定义

| 数据类型 | 开始时间 | 结束时间 |
|----------|----------|----------|
| 打车（滴滴） | 开始计费时间 | 结束计费时间 |
| 打车（美团） | 行程开始时间 | 行程到达时间 |
| 酒店 | 入住时间 | 离店时间（可能跨天） |
| 机票 | 起飞时间 | 到达时间 |
| 无明确时间 | 费用发生日期 00:00 | 费用发生日期 23:59 |

#### 4.2.3 时间精度
- 精确到**小时**
- 判断条件：`开始时间A < 结束时间B 且 开始时间B < 结束时间A`

### 4.3 地点冲突检测

#### 4.3.1 检测逻辑
1. 按出差人分组
2. 对每人在同一时间区间内的所有记录提取地点信息
3. 检查同一时间区间内是否出现在不同地点
4. 若地点不同 → 标记为地点冲突异常

#### 4.3.2 地点精度

| 数据类型 | 地点精度 | 提取规则 |
|----------|----------|----------|
| 打车（滴滴） | 区县级 | 出发区县、到达区县（取非空值） |
| 打车（美团） | 区县级 | 从地点文本解析区县 |
| 酒店 | 区县级 | 从酒店地址解析区县 |
| 机票 | **城市级** | 出发城市、到达城市 |

#### 4.3.3 混合精度处理
- 机票使用城市级别检测
- 其他类型使用区县级别检测
- 当机票与打车/酒店时间重叠时，使用城市级别对比
- 当打车与酒店时间重叠时，使用区县级别对比

### 4.4 多地行程警示

#### 4.4.1 检测逻辑
1. 按出差人分组
2. 按日期聚合每人的所有记录
3. 统计当天出现的城市数量
4. 若城市数量 > 1 → 标记为多地行程警示（黄色）

#### 4.4.2 说明
- 此情况通常为正常出差（如转机、跨城市出差）
- 不算异常，但需要人工确认
- 用黄色标记以区分

### 4.5 异常标记方式

| 异常类型 | 标记方式 | Excel样式 |
|----------|----------|-----------|
| 时间重叠异常 | 整行标红 | 背景色：#FF0000，字体：白色 |
| 地点冲突异常 | 整行标红 | 背景色：#FF0000，字体：白色 |
| 多地行程警示 | 整行标黄 | 背景色：#FFFF00，字体：黑色 |

同时，在核对报告中详细列出所有异常记录。

## 五、核对机制

### 5.1 核对目标
防止程序处理过程中出现数据丢失、转换错误等问题，确保数据完整性。

### 5.2 数据完整性核对

#### 5.2.1 输入核对
```
核对项目：
1. 文件读取核对
   ✓ 滴滴文件存在且可读
   ✓ 美团文件存在且可读
   ✓ 在途文件存在且可读

2. 数据量核对
   ✓ 滴滴订单数 = 实际读取行数
   ✓ 美团各Sheet数据量 = 实际读取行数
   ✓ 在途订单数 = 实际读取行数
```

#### 5.2.2 转换核对
```
核对项目：
1. 条数核对
   ✓ 输入总条数 = 各源数据条数之和
   ✓ 输出总条数 = 输入总条数（无丢失）

2. 字段核对
   ✓ 必填字段（出差人、费用发生日期）无空值
   ✓ 日期字段格式正确
   ✓ 金额字段为数值或空

3. 映射成功率
   ✓ 出差人映射成功率 > 99%
   ✓ 时间字段映射成功率 > 95%
   ✓ 地点字段映射成功率 > 90%
```

#### 5.2.3 异常检测核对
```
核对项目：
1. 检测覆盖率
   ✓ 已检测记录数 = 总记录数
   ✓ 检测异常记录数 = 标红记录数

2. 异常类型核对
   ✓ 时间重叠异常数量
   ✓ 地点冲突异常数量
   ✓ 多地行程警示数量
```

### 5.4 数据校验点

| 校验项 | 校验规则 | 失败处理 |
|--------|----------|----------|
| 出差人 | 非空且非空白字符串 | 记录警告，该条不计入统计 |
| 费用发生日期 | 有效日期格式 (YYYY-MM-DD 或 Excel 序列号) | 记录警告，跳过该条 |
| 时间字段 | 有效时间格式或可解析 | 记录警告，时间区间置空 |
| 金额字段 | 数值类型或空值 | 记录警告，金额置 0 |
| 地点字段 | 非空字符串 | 记录警告，标记为"未知地点" |

namespace TravelExpenseAnalyzer.Models;

public class TravelRecord
{
    // ========== 目标字段（输出到Excel）==========
    public string 出差人 { get; set; }

    public string 交通工具 { get; set; }

    public decimal? 交通费税额 { get; set; }

public enum DataSourceType
