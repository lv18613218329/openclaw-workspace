## Context

当前电子白板应用使用Vue 3和Konva.js进行图形绘制和管理，支持基本的图形绘制（如矩形、圆形、线条等）和编辑功能。然而，用户无法对已有的图形进行高级的组合和修改操作，如并集、交集等布尔运算。

本次设计旨在添加图形布尔运算功能，支持用户选择两个图形并执行四种基本的布尔运算操作：并集(Union)、交集(Intersection)、差集(Difference)和分割(Clip)。

## Goals / Non-Goals

**Goals:**
- 实现图形多选功能，支持选择两个图形进行布尔运算
- 集成Clipper2库进行精确的路径计算
- 提供直观的UI界面，包括运算类型选择和执行按钮
- 正确处理运算结果的图形创建和原图形删除
- 实现异常处理机制，提示无效操作和错误信息

**Non-Goals:**
- 不支持三个或更多图形的批量布尔运算
- 不修改现有的图形绘制和基本编辑功能
- 不涉及性能优化（如大型复杂图形的处理）

## Decisions

### 1. 图形选择机制
- **Decision**: 使用Konva.js的内置多选功能，通过Shift+点击实现
- **Rationale**: Konva.js已经提供了完善的图形选择API，使用内置功能可以减少开发工作量，保证与现有系统的兼容性
- **Alternative**: 实现自定义选择机制 - 被拒绝，因为会增加复杂性和潜在的bug

### 2. 布尔运算库选择
- **Decision**: 集成Clipper2库
- **Rationale**: Clipper2是一个成熟的开源几何计算库，支持精确的多边形布尔运算，性能良好且文档丰富
- **Alternative**: 使用其他几何库（如Turf.js）- 被拒绝，因为Turf.js主要面向地理空间数据，不适用于通用图形布尔运算

### 3. UI实现方式
- **Decision**: 在工具栏添加布尔运算专门区域，包含运算类型选择和执行按钮
- **Rationale**: 这种方式直观易用，符合用户的操作习惯，与现有工具栏布局保持一致
- **Alternative**: 使用上下文菜单 - 被拒绝，因为需要额外的点击操作，不如直接在工具栏操作方便

### 4. 运算结果处理
- **Decision**: 创建新图形并删除原图形
- **Rationale**: 这种方式符合用户对布尔运算的预期，保持画布整洁
- **Alternative**: 保留原图形并创建新图形 - 被拒绝，因为会导致画布上图形过多，增加用户管理负担

## Risks / Trade-offs

### [Risk] 图形不相交导致运算失败
- **Mitigation**: 实现图形相交检测，在执行运算前检查，如果不相交则显示提示信息，不执行运算

### [Risk] 复杂图形运算性能问题
- **Mitigation**: 限制单次运算的图形复杂度，对于过于复杂的图形显示提示信息

### [Risk] 运算结果图形属性继承问题
- **Mitigation**: 制定属性继承规则（如继承第一个选择图形的颜色、填充等属性），并在UI中向用户说明

### [Risk] 用户误操作
- **Mitigation**: 添加确认对话框，在执行运算前提示用户，允许取消操作
