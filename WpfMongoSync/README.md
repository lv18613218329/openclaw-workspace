# MongoDB → 飞书多维表格 同步工具

一个WPF桌面应用程序，用于从MongoDB查询数据并同步到飞书多维表格。支持增量同步模式，同步完成后可自动调用 Coze 智能体。

## 功能特性

- ✅ MongoDB数据查询（支持自定义过滤条件）
- ✅ **增量同步** - 只同步新增数据，不删除已有记录
- ✅ **时间范围查询** - 按 `events.local_time_ms` 时间戳过滤
- ✅ **长查询支持** - 查询超时时间5分钟，支持大数据量查询
- ✅ **定时自动同步** - 每天指定时间自动增量同步
- ✅ **Coze 智能体集成** - 同步完成后自动调用扣子智能体
- ✅ 飞书多维表格同步（支持追加模式）
- ✅ 配置保存
- ✅ 数据预览
- ✅ 同步状态记录

## 技术栈

- .NET 8.0 Windows
- WPF
- MongoDB Driver
- Newtonsoft.Json

## 使用说明

### 第一步：配置MongoDB连接

- **连接字符串**：MongoDB连接URL（已预填）
- **数据库**：数据库名称（默认 `yn_collect`）
- **集合**：集合名称（默认 `wjs_track_data`）
- **事件类型**：`events.event` 字段的过滤值（默认 `MQConfigError`）
- **租户代码**：`header.custom.tenantCode` 字段的过滤值（可选）

> ⏱️ **查询超时**：默认5分钟，适合大数据量查询。如果仍超时，可在代码中调整 `DefaultQueryTimeoutSeconds`。

### 第二步：查询数据

提供两种查询模式：

1. **🔍 查询全部** - 查询所有符合条件的数据
2. **🕐 查询24小时** - 只查询最近24小时的数据（用于预览）

### 第三步：配置飞书

#### 3.1 创建飞书应用

1. 访问 [飞书开放平台](https://open.feishu.cn/)
2. 创建企业自建应用
3. 获取 `App ID` 和 `App Secret`

#### 3.2 配置应用权限

在飞书应用后台，添加以下权限：
- `bitable:record` - 查看、评论、编辑多维表格数据
- `bitable:record:read` - 获取多维表格数据

#### 3.3 创建多维表格

1. 在飞书中创建一个多维表格
2. **创建以下三个字段**（字段名必须完全一致）：

| 字段名 | 字段类型 | 说明 |
|-------|---------|------|
| **异常名称** | 文本 | 事件类型（如 MQConfigError） |
| **时间** | 文本 | 事件发生时间（格式：yyyy-MM-dd HH:mm:ss） |
| **内容** | 多行文本 | 完整的原始JSON数据 |

3. 获取多维表格的 `App Token` 和 `Table ID`

**如何获取App Token和Table ID：**
- 打开多维表格网页版
- URL格式：`https://xxx.feishu.cn/base/[AppToken]?table=[TableId]`

### 第四步：同步数据

提供三种同步模式：

1. **📤 全量同步** - 清空目标表格后重新写入所有数据
2. **➕ 增量同步** - 只追加新增数据，不清空已有记录
3. **⏰ 定时同步** - 每天指定时间自动执行增量同步（默认早上6点）

### 第五步：配置 Coze 智能体（可选）

同步完成后可自动调用 Coze（扣子）智能体 API：

1. **Access Token** - 个人访问令牌（在扣子开放平台获取）
2. **Bot ID** - 智能体的 Bot ID
3. **查询内容** - 发送给智能体的消息内容

> 触发时机：增量同步完成后（无论成功/失败/无数据）都会调用

## 字段映射说明

程序会自动将MongoDB查询结果转换为飞书表格的三列：

| 飞书表格列名 | 数据来源 |
|------------|---------|
| **异常名称** | 界面上配置的事件类型（EventFilter） |
| **时间** | MongoDB 数据中的 `local_time` 字段（自动转换为 `yyyy-MM-dd HH:mm:ss` 格式） |
| **内容** | 完整的原始JSON数据（格式化后的） |

## 增量同步说明

### 工作原理

1. 按 `events.local_time_ms` 时间戳字段过滤数据
2. 记录最后同步时间，下次只查询该时间之后的新数据
3. 数据追加到飞书表格，不删除已有记录
4. 同步状态保存在 `%APPDATA%\WpfMongoSync\sync_state.json`

### 查询条件

```javascript
{
  "events.event": "MQConfigError",           // 事件类型
  "events.local_time_ms": { $gt: 1710000000000 }  // 时间戳大于上次同步时间
}
```

### 同步状态

程序会显示：
- 上次同步时间
- 累计同步记录数

点击 **🔄 重置状态** 可以重置同步状态，下次将从最近24小时开始同步。

## 定时同步

1. 点击 **⏰ 定时同步** 按钮
2. 设置每天执行时间（默认 6:00）
3. 程序会在后台等待，到时间自动执行增量同步

**注意**：程序需要保持运行状态才能执行定时同步。

## 常见问题

### Q: 查询数据没有结果？

1. **检查过滤条件**：确认 `events.event` 字段值是否正确
2. **检查字段路径**：确认 `events.local_time_ms` 和 `header.custom.tenantCode` 路径是否正确
3. **检查索引**：MongoDB大数据量查询可能需要创建索引
4. **等待时间**：大数据量查询可能需要100-200秒，程序已支持5分钟超时

### Q: 增量同步重复数据？

增量同步基于 `events.local_time_ms` 时间戳。如果数据的时间戳早于最后同步时间，可能会漏掉；如果时间戳有更新，可能会重复。建议在飞书表格中设置唯一字段去重。

### Q: 如何从头开始同步？

点击 **🔄 重置状态** 按钮，然后点击 **➕ 增量同步**，会从最近24小时开始同步。

### Q: 定时同步不执行？

确保程序保持运行状态。如果程序关闭，定时同步将不会执行。

### Q: 飞书表格列名不对？

确保飞书多维表格中的列名完全一致：
- `异常名称`（不是"问题名称"）
- `时间`
- `内容`
