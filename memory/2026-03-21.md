## MongoDB 同步项目 (WpfMongoSync)

### 项目位置
`C:\Users\Administrator\.openclaw\workspace\WpfMongoSync`

### 项目功能
MongoDB → 飞书多维表格 同步工具，WPF 桌面应用

### 核心功能
1. **MongoDB 查询** - 支持 `events.event` 和 `header.custom.tenantCode` 过滤
2. **增量同步** - 按 `events.local_time_ms` 时间戳过滤，只追加新数据
3. **全量同步** - 清空飞书表格后重新写入
4. **定时同步** - 每天指定时间自动执行增量同步
5. **Coze 智能体集成** - 同步完成后自动调用扣子智能体 API
6. **同步记录显示** - 界面显示最近 10 条同步历史

## 出差日记账数据整合工具

### 项目位置
`C:\Users\Administrator\.openclaw\workspace\TravelExpenseAnalyzer`

### 项目背景
追溯和还原员工的出行轨迹和费用情况，核实员工是否同一时间出现在多个地点和多笔费用的违规行为。

### 数据来源
| 文件 | 数据量 | 说明 |
|------|--------|------|
| 滴滴用车订单 | 281 条 | 打车行程 |
| 美团商旅订单 | 497 条 | 打车118 + 酒店350 + 机票22 + 餐饮7 |
| 在途差旅订单 | 497 条 | 酒店/机票/服务 |

### 核心功能
1. **数据整合** - 将滴滴、美团、在途数据整合成统一格式
2. **异常检测** - 时间重叠（红）、地点冲突（红）、多地行程（黄）
3. **核对机制** - 数据完整性核对，生成核对报告

### 异常检测规则
| 类型 | 定义 | 颜色 |
|------|------|------|
| 时间重叠异常 | 同一人同一时间在不同地点 | 红色 |
| 地点冲突异常 | 同一人同一时间在不同区县 | 红色 |
| 多地行程警示 | 一天内去多个城市 | 黄色 |

### 检测精度
- 时间重叠：精确到小时
- 地点冲突：打车/酒店用区县级，机票用城市级

### 输出文件
- `出差日记账_生成_{日期}.xlsx` - 整合后的数据（异常行标红/黄）
- `核对报告_{日期}.txt` - 数据核对报告

### 文档

- `技术设计文档.md` - 技术实现设计
